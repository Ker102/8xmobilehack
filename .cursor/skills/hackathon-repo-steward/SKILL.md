# Hackathon Repo Steward

## Checklist

1. Read `README.md`, `docs/ARCHITECTURE.md`, and `docs/CASE_STUDY.md`.
2. Keep the user journey and architecture understandable to judges.
3. Add or update validation for meaningful behavior changes.
4. Keep secrets out of files and use environment variables.
5. Run `npm run validate`.
6. Summarize what changed in product, architecture, validation, and demo readiness terms.
